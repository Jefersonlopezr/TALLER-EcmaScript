¿Qué es ECMAScript?

ECMAScript es el estándar creado por ECMA International para definir cómo debe funcionar el lenguaje JavaScript. Gracias a esta especificación, todos los navegadores y entornos pueden ejecutar código JavaScript de manera coherente.

JavaScript es el lenguaje basado en esta especificación. Las nuevas versiones de ECMAScript introducen mejoras de sintaxis, nuevas funcionalidades, y correcciones que hacen que el desarrollo web sea más potente y moderno.

¿Cómo se utiliza ECMAScript?

ECMAScript se usa escribiendo JavaScript moderno en navegadores o editores. Las funciones más recientes pueden necesitar herramientas como Babel o Webpack para ser compatibles con navegadores antiguos.

Versiones de ECMAScript

🔹 ECMAScript 1 (1997)

Primera version estandarizada del lenguaje.

// Ejemplo básico de ES1
var x = 10;
if (x > 5) {
  alert('x es mayor que 5');
}

🔹 ECMAScript 2 (1998)

Pequeñas actualizaciones para alinear con el estandar ISO.

🔹 ECMAScript 3 (1999)

  • Expresiones Regulares.
  • Manejo de excepciones.
  • Mejoras en operadores.
// Try/Catch en ES3
try {
  undefinedFunction();
} catch(e) {
  console.log('Error: ' + e.message);
}

🔹 ECMAScript 5 (2009)

Incluye strict mode, metodos de arrays como forEach y JSON. 

"use strict";
[1, 2, 3].forEach(function(n) {
  console.log(n);
});

🔹 ES6 / ES2015

Una de las mayores actualizaciones. Introdujo let, const, funciones flecha, clases, template strings y más. 

const nombre = "Juan";
console.log(`Hola, ${nombre}`);
const suma = (a, b) => a + b;

🔹ES2016 (ES7)

Se agregaron el operador de exponenciación y includes en arrays. 

console.log(2 ** 3); // 8
console.log([1, 2, 3].includes(2)); // true

🔹 ES2017 (ES8)

Introdujo async/await y Object.entries.

async function saludar() {
  return "Hola";
}
saludar().then(console.log);

🔹 ES2018 (ES9)

Soporte para rest/spread en objetos y método finally en promesas. 

const obj = { a: 1, b: 2 };
const copia = { ...obj };

🔹 ES2019

Se agregaron flat y flatMap a los arrays. 

[1, [2, [3]]].flat(2); // [1, 2, 3]

🔹 ES2020

Incluyó Optional Chaining y Nullish Coalescing. 

const usuario = {};
console.log(usuario?.perfil?.nombre ?? "Desconocido");

🔹 ES2021

Agregó Logical Assignment y String.replaceAll. 

let texto = "hola mundo mundo";
console.log(texto.replaceAll("mundo", "🌍"));

🔹 ES2022

Incluye at() para arrays y soporte para top-level await. 

const letras = ['a', 'b', 'c'];
console.log(letras.at(-1)); // "c"

🔹 ES2023

Nuevas funciones en Array como findLast y mejoras en Symbol. 

const arr = [1, 2, 3, 4];
console.log(arr.findLast(n => n < 4)); // 3

🔹 ES2024 (Propuesta)

Se esperan mejoras en Observables y Temporal para manejo de fechas.

Diferencias notorias entre versiones

Versión Caracteristicas clave
ES5 Strict mode, JSON, forEach
ES6 let/const, arrow functions, clases, import/export
ES7-S10 includes, exponentiation, async/await, spread
ES11-S14 Optional chaining, at(), top-level await

Conclusión

ECMAScript evoluciona constantemente, añadiendo características que hacen a JavaScript más potente y fácil de usar. Mantenerse actualizado con las nuevas versiones permite escribir código más limpio y eficiente.
